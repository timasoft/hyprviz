## Диспетчеры

`layoutmsg` команды:

| команда | описание | параметры |
| --- | --- | --- |
| swapwithmaster | меняет местами текущее окно с мастер-окном. Если текущее окно является мастером, меняет его местами с первым дочерним окном. | либо `master` (новый фокус на новом мастер-окне), `child` (новый фокус на новом дочернем окне) или `auto` (по умолчанию; сохраняет фокус на ранее сфокусированном окне). Добавление `ignoremaster` игнорирует этот диспетчер, если мастер уже в фокусе. |
| focusmaster | фокусируется на мастер-окне. | либо `master` (фокус остаётся на мастер-окне), `auto` (по умолчанию; фокус на первом не-мастер окне, если уже на мастер-окне) или `previous` (запоминает текущее окно при фокусировке на мастер, если уже на мастер-окне, фокус на предыдущем или откат к `auto`). |
| cyclenext | фокусируется на следующем окне с учётом раскладки | либо `loop` (разрешает зацикливание от нижней части стека обратно к мастеру) или `noloop` (принудительная остановка внизу стека, как в DWM). `loop` используется по умолчанию, если не указано. |
| cycleprev | фокусируется на предыдущем окне с учётом раскладки | либо `loop` (разрешает зацикливание от мастера к нижней части стека) или `noloop` (принудительная остановка на мастере, как в DWM). `loop` используется по умолчанию, если не указано. |
| swapnext | меняет местами сфокусированное окно со следующим окном с учётом раскладки | либо `loop` (разрешает обмен нижней частью стека и мастером) или `noloop` (не разрешает, как в DWM). `loop` используется по умолчанию, если не указано. |
| swapprev | меняет местами сфокусированное окно с предыдущим окном с учётом раскладки | либо `loop` (разрешает обмен мастером и нижней частью стека) или `noloop` (не разрешает, как в DWM). `loop` используется по умолчанию, если не указано. |
| addmaster | добавляет мастер на мастер-сторону. Активным окном будет текущее окно, если оно не мастер, или первое не-мастер окно. | нет |
| removemaster | удаляет мастер с мастер-стороны. Активным окном будет текущее окно, если оно мастер, или последнее мастер-окно. | нет |
| orientationleft | устанавливает ориентацию для текущего рабочего пространства влево (мастер-область слева, подчинённые окна справа, уложены вертикально) | нет |
| orientationright | устанавливает ориентацию для текущего рабочего пространства вправо (мастер-область справа, подчинённые окна слева, уложены вертикально) | нет |
| orientationtop | устанавливает ориентацию для текущего рабочего пространства вверх (мастер-область сверху, подчинённые окна снизу, уложены горизонтально) | нет |
| orientationbottom | устанавливает ориентацию для текущего рабочего пространства вниз (мастер-область снизу, подчинённые окна сверху, уложены горизонтально) | нет |
| orientationcenter | устанавливает ориентацию для текущего рабочего пространства по центру (мастер-область по центру, подчинённые окна чередуются слева и справа, уложены вертикально) | нет |
| orientationnext | циклически переключается на следующую ориентацию для текущего рабочего пространства (по часовой стрелке) | нет |
| orientationprev | циклически переключается на предыдущую ориентацию для текущего рабочего пространства (против часовой стрелки) | нет |
| orientationcycle | циклически переключается на следующую ориентацию из предоставленного списка для текущего рабочего пространства | допустимые значения: `left`, `top`, `right`, `bottom` или `center`. Значения должны быть разделены пробелом. Если оставить пустым, будет работать как `orientationnext` |
| mfact | изменяет mfact, коэффициент разделения мастер-области | новое соотношение, относительная дельта с плавающей точкой (напр. `-0.2` или `+0.2`) или `exact` с последующим точным значением с плавающей точкой между 0.0 и 1.0 |
| rollnext | поворачивает следующее окно в стеке, чтобы оно стало мастером, сохраняя фокус на мастер-окне | нет |
| rollprev | поворачивает предыдущее окно в стеке, чтобы оно стало мастером, сохраняя фокус на мастер-окне | нет |

Параметры для команд разделяются одним пробелом.

> [!NOTE]
> Пример использования:
> 
> ```ini
> bind = MOD, KEY, layoutmsg, cyclenext
> # ведёт себя как функция promote в xmonad (https://hackage.haskell.org/package/xmonad-contrib-0.17.1/docs/XMonad-Actions-Promote.html)
> bind = MOD, KEY, layoutmsg, swapwithmaster master
> ```

## Правила рабочих пространств

`layoutopt` правила:

| правило | описание | тип |
| --- | --- | --- |
| orientation:[o] | Устанавливает ориентацию рабочего пространства. Доступные ориентации см. в [Конфигурация->ориентация](#config) | строка |

Пример использования:

```ini
workspace = 2, layoutopt:orientation:top
```
