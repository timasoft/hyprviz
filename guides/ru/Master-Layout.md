## Диспетчеры

Команды `layoutmsg`:

| команда | описание | параметры |
| --- | --- | --- |
| swapwithmaster | меняет местами текущее окно с мастер-окном. Если текущее окно является мастером, меняет его местами с первым дочерним окном. | либо `master` (новый фокус на новом мастер-окне), `child` (новый фокус на новом дочернем окне) или `auto` (по умолчанию; сохраняет фокус на ранее активном окне). Добавление `ignoremaster` игнорирует этот диспетчер, если мастер уже в фокусе. |
| focusmaster | фокусируется на мастер-окне. | либо `master` (фокус остается на мастер-окне), `auto` (по умолчанию; фокусируется на первом не-мастер окне, если уже на мастер-окне) или `previous` (запоминает текущее окно при фокусировке на мастер-окне, если уже на мастер-окне, фокусируется на предыдущем или возвращается к `auto`). |
| cyclenext | фокусируется на следующем окне с учетом макета | либо `loop` (разрешает циклический переход от нижнего окна обратно к мастеру) или `noloop` (принудительно останавливается внизу стека, как в DWM). `loop` используется по умолчанию, если не указано. |
| cycleprev | фокусируется на предыдущем окне с учетом макета | либо `loop` (разрешает циклический переход от мастера к нижнему окну) или `noloop` (принудительно останавливается на мастере, как в DWM). `loop` используется по умолчанию, если не указано. |
| swapnext | меняет местами активное окно со следующим окном с учетом макета | либо `loop` (разрешает обмен нижним окном и мастером) или `noloop` (не разрешает, как в DWM). `loop` используется по умолчанию, если не указано. |
| swapprev | меняет местами активное окно с предыдущим окном с учетом макета | либо `loop` (разрешает обмен мастером и нижним окном) или `noloop` (не разрешает, как в DWM). `loop` используется по умолчанию, если не указано. |
| addmaster | добавляет мастер к мастер-стороне. Это будет активное окно, если оно не мастер, или первое не-мастер окно. | нет |
| removemaster | удаляет мастер из мастер-стороны. Это будет активное окно, если оно мастер, или последнее мастер-окно. | нет |
| orientationleft | устанавливает ориентацию для текущего рабочего пространства влево (мастер-область слева, дочерние окна справа, вертикально уложенные) | нет |
| orientationright | устанавливает ориентацию для текущего рабочего пространства вправо (мастер-область справа, дочерние окна слева, вертикально уложенные) | нет |
| orientationtop | устанавливает ориентацию для текущего рабочего пространства вверх (мастер-область сверху, дочерние окна снизу, горизонтально уложенные) | нет |
| orientationbottom | устанавливает ориентацию для текущего рабочего пространства вниз (мастер-область снизу, дочерние окна сверху, горизонтально уложенные) | нет |
| orientationcenter | устанавливает ориентацию для текущего рабочего пространства по центру (мастер-область в центре, дочерние окна чередуются слева и справа, вертикально уложенные) | нет |
| orientationnext | циклически переключается к следующей ориентации для текущего рабочего пространства (по часовой стрелке) | нет |
| orientationprev | циклически переключается к предыдущей ориентации для текущего рабочего пространства (против часовой стрелки) | нет |
| orientationcycle | циклически переключается к следующей ориентации из предоставленного списка для текущего рабочего пространства | допустимые значения: `left`, `top`, `right`, `bottom` или `center`. Значения должны быть разделены пробелом. Если оставить пустым, будет работать как `orientationnext` |
| mfact | изменяет mfact, соотношение мастер-разделения | новое соотношение, относительная дельта с плавающей запятой (например `-0.2` или `+0.2`) или `exact` с последующим точным значением с плавающей запятой между 0.0 и 1.0 |
| rollnext | вращает следующее окно в стеке, чтобы стать мастером, сохраняя фокус на мастер-окне | нет |
| rollprev | вращает предыдущее окно в стеке, чтобы стать мастером, сохраняя фокус на мастер-окне | нет |

Параметры для команд разделяются одним пробелом.

{{< callout type=info >}}

Пример использования:

```ini
bind = MOD, KEY, layoutmsg, cyclenext
# ведет себя как функция promote в xmonad (https://hackage.haskell.org/package/xmonad-contrib-0.17.1/docs/XMonad-Actions-Promote.html)
bind = MOD, KEY, layoutmsg, swapwithmaster master
```

{{< /callout >}}

## Правила рабочих пространств

Правила `layoutopt`:

| правило | описание | тип |
| --- | --- | --- |
| orientation:[o] | Устанавливает ориентацию рабочего пространства. Доступные ориентации см. в разделе [Config->orientation](#config) | строка |

Пример использования:

```ini
workspace = 2, layoutopt:orientation:top
```
