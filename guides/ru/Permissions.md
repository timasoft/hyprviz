> [!NOTE]
> Переведено с en с помощью Qwen3.5-Plus

Если у вас установлен `hyprland-guiutils`, вы можете использовать встроенную
систему разрешений Hyprland.

На данный момент доступно лишь несколько разрешений, но в будущем их может стать больше.

## Разрешения

Разрешения работают примерно как в Android. Если приложение пытается выполнить что-то чувствительное с
композитором (Hyprland), Hyprland покажет уведомление с вопросом, хотите ли вы
разрешить ему это сделать.

> [!NOTE]
> Перед настройкой разрешений убедитесь, что вы включили их, установив
> `ecosystem:enforce_permissions = true`, так как по умолчанию они отключены.


### Настройка разрешений

> [!IMPORTANT]
> Разрешения, настроенные в конфиге, **не** перезагружаются на лету и требуют
> перезапуска Hyprland по соображениям безопасности.

Настроить их просто:

```ini
permission = regex, permission, mode
```

например:
```ini
permission = /usr/bin/grim, screencopy, allow
```
Это позволит `/usr/bin/grim` всегда захватывать ваш экран без запроса.

```ini
permission = /usr/bin/appsuite-.*, screencopy, allow
```
Это позволит любому приложению, путь которого начинается с `/usr/bin/appsuite-`, захватывать ваш экран без запроса.


### Режимы разрешений

Доступно 3 режима:
- `allow`: Не спрашивать, просто разрешить приложению продолжить.
- `ask`: Показывать уведомление каждый раз, когда приложение пытается выполнить что-то чувствительное. Эти всплывающие окна позволяют вам: Отклонить, Разрешить до выхода приложения или Разрешить до выхода Hyprland.
- `deny`: Не спрашивать, всегда запрещать доступ приложению.


### Список разрешений

`screencopy`:
 - По умолчанию: **ASK**
 - Доступ к вашему экрану _без_ использования xdg-desktop-portal-hyprland. Примеры: `grim`, `wl-screenrec`, `wf-recorder`.
 - Если отклонено, будет отображён чёрный экран с текстом "permission denied".
 - Зачем запрещать? Для приложений / скриптов, которые могут злонамеренно попытаться захватить ваш экран без вашего ведома, используя протоколы wayland напрямую.

`plugin`:
 - По умолчанию: **ASK**
 - Доступ к загрузке плагина. Может быть либо regex для бинарного файла приложения, либо путь к плагину.
 - _Не_ разрешайте `hyprctl` загружать ваши плагины по умолчанию (злоумышленник может выполнить `hyprctl plugin load /tmp/my-malicious-plugin.so`) — используйте либо `deny` для отключения, либо `ask` для получения запроса.

`keyboard`:
 - По умолчанию: **ALLOW**
 - Доступ к подключению новой клавиатуры. Regex имени устройства.
 - Если вы хотите отключить все клавиатуры, не соответствующие regex, создайте правило, устанавливающее `DENY` для `.*` _в качестве последнего правила разрешения для клавиатур_.
 - Зачем запрещать? Rubber duckies, злонамеренные виртуальные / usb-клавиатуры.

## Примечания

Реализации **xdg-desktop-portal** (включая xdph) — это просто обычные приложения. Они также будут проходить через систему разрешений. Возможно, вы захотите добавить правило вида:
```ini
permission = /usr/(lib|libexec|lib64)/xdg-desktop-portal-hyprland, screencopy, allow
```
если вы не разрешаете screencopy для всех приложений.

<br/>

В NixOS нет статических путей для бинарных файлов, поэтому необходимо использовать regex. Эти примеры правил позволяют `grim` и `xdg-desktop-portal-hyprland` копировать экран:
```ini
permission = /nix/store/[a-z0-9]{32}-grim-[0-9.]*/bin/grim, screencopy, allow
permission = /nix/store/[a-z0-9]{32}-xdg-desktop-portal-hyprland-[0-9.]*/libexec/.xdg-desktop-portal-hyprland-wrapped, screencopy, allow
```

При рендеринге конфигурации с помощью самого Nix также можно использовать интерполяцию строк (учтите, что если путь содержит специальные символы regex (например, `+`), их необходимо экранировать):
```ini
permission = ${lib.getExe pkgs.grim}, screencopy, allow
permission = ${lib.escapeRegex (lib.getExe config.programs.hyprlock.package)}, screencopy, allow
permission = ${pkgs.xdg-desktop-portal-hyprland}/libexec/.xdg-desktop-portal-hyprland-wrapped, screencopy, allow
```

<br/>

В некоторых системах **BSD** пути могут не работать. В таких случаях вы можете полностью отключить разрешения, установив
```ini
ecosystem {
  enforce_permissions = false
}
```
в противном случае у вас не будет _конфигурационного_ управления разрешениями (всплывающие окна всё ещё будут работать, хотя и не будут показывать пути, а опция "remember" будет недоступна).
