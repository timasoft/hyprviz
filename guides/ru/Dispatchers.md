## Объяснение параметров

| Тип параметра | Описание |
| --- | --- |
| window | окно. Любой из следующих вариантов: регулярное выражение класса (по умолчанию, необязательно `class:`), `initialclass:` начальное регулярное выражение класса, `title:` регулярное выражение заголовка, `initialtitle` начальное регулярное выражение заголовка, `tag:` регулярное выражение тега окна, `pid:` PID процесса, `address:` адрес, `activewindow` активное окно, `floating` первое плавающее окно на текущем рабочем пространстве, `tiled` первое плиточное окно на текущем рабочем пространстве |
| workspace | см. [ниже]({{< relref "#workspaces" >}}). |
| direction | `l` `r` `u` `d` влево вправо вверх вниз |
| monitor | Один из: направление, ID, имя, `current`, относительное (например, `+1` или `-1`) |
| resizeparams | относительный вектор дельты в пикселях (например, `10 -10`), необязательно процент от размера окна (например, `20 25%`) или `exact` с последующим точным вектором (например, `exact 1280 720`), необязательно процент от размера экрана (например, `exact 50% 50%`) |
| floatvalue | относительная дельта с плавающей запятой (например, `-0.2` или `+0.2`) или `exact` с последующим точным значением (например, `exact 0.5`) |
| zheight | `top` или `bottom` |
| mod | `SUPER`, `SUPER_ALT` и т.д. |
| key | `g`, `code:42`, `42` или клики мыши (`mouse:272`) |

> [!WARNING]
> Пользователям [uwsm](../../Useful-Utilities/Systemd-start) следует избегать использования диспетчера `exit` или прямого завершения процесса Hyprland, так как это удаляет композитор из-под управляемых клиентов и нарушает упорядоченную последовательность завершения сессии. Используйте `exec, uwsm stop` (или [другие варианты](https://github.com/Vladimir-csp/uwsm#how-to-stop)), чтобы корректно завершить графическую сессию (и привязанную к ней сессию входа, если применимо). При возникновении проблем с несогласованными состояниями юнитов, влияющими на последующие сессии, используйте `exec, loginctl terminate-user ""` (завершает все юниты пользователя).
>
> Также настоятельно рекомендуется заменить диспетчер `exit` в разделе привязок файла `hyprland.conf` на указанные альтернативы.

> [!WARNING]
> Устанавливать DPMS или forceidle напрямую через привязку клавиш НЕ рекомендуется, так как это
> может вызвать неопределённое поведение. Вместо этого рассмотрите вариант:
>
> ```ini
> bind = MOD, KEY, exec, sleep 1 && hyprctl dispatch dpms off
> ```

### Группированные (вкладочные) окна

Hyprland позволяет создать группу из текущего активного окна с помощью
диспетчера `togglegroup`.

Группа аналогична «вкладочному» контейнеру в i3wm. Она занимает место одного окна,
и вы можете переключаться между окнами в группе с помощью диспетчера
`changegroupactive`.

Цвета границ новой группы настраиваются соответствующими параметрами `col.`
в секции конфигурации `group`.

Группу можно заблокировать с помощью диспетчера `lockactivegroup`, чтобы предотвратить
добавление новых окон в эту группу. Диспетчер `lockgroups` позволяет включить/выключить
глобальную блокировку всех групп, независимо от их локального состояния.

Запретить окну присоединяться к группе или становиться группой можно с помощью
диспетчера `denywindowfromgroup`. В этом случае `movewindoworgroup` будет вести себя
как `movewindow`, если текущее активное окно или окно в указанном направлении имеет
данное свойство.

## Рабочие пространства

У вас есть девять вариантов:

- ID: например, `1`, `2` или `3`

- Относительный ID: например, `+1`, `-3` или `+100`

- рабочее пространство на мониторе, относительное с `+` или `-`, абсолютное с `~`: например, `m+1`,
  `m-2` или `m~3`

- рабочее пространство на мониторе, включая пустые, относительное с `+` или `-`,
  абсолютное с `~`: например, `r+1` или `r~3`

- открытое рабочее пространство, относительное с `+` или `-`, абсолютное с `~`: например, `e+1`,
  `e-10` или `e~2`

- Имя: например, `name:Web`, `name:Anime` или `name:Better anime`

- Предыдущее рабочее пространство: `previous`, или `previous_per_monitor`

- Первое доступное пустое рабочее пространство: `empty`; суффикс `m` — поиск только на текущем мониторе,
  и/или `n` — следующее доступное пустое. Например: `emptynm`

- Специальное рабочее пространство: `special` или `special:name` для именованных специальных рабочих пространств.

> [!WARNING]
> `special` поддерживается ТОЛЬКО в `movetoworkspace` и `movetoworkspacesilent`.
> Любой другой диспетчер приведёт к недокументированному поведению.

> [!WARNING]
> Числовые рабочие пространства (например, `1`, `2`, `13371337`) разрешены **ТОЛЬКО** в диапазоне от 1
> до 2147483647 (включительно).
> Ни `0`, ни отрицательные числа не допускаются.

## Специальное рабочее пространство

Специальное рабочее пространство — это аналог «скрэтчпада» в других средах: рабочее пространство,
которое можно включать/выключать на любом мониторе.

> [!NOTE]
> Можно определить несколько именованных специальных рабочих пространств, но их количество
> ограничено 97 одновременно.

Пример перемещения окна/приложения на специальное рабочее пространство:

```ini
bind = SUPER, C, movetoworkspace, special
# Приведённая выше команда переместит окно на специальное рабочее пространство при нажатии 'SUPER'+'C'.
# Чтобы отобразить скрытое окно, используйте диспетчер `togglespecialworkspace`, упомянутый выше.
```

## Выполнение с правилами

Диспетчер `exec` поддерживает добавление правил. Учтите: поведение может различаться
в зависимости от окна. Hyprland отслеживает PID запущенного процесса. Если процесс
порождает дочерний процесс, который открывает окно — правило может не сработать.

Синтаксис:

```ini
bind = mod, key, exec, [правила...] команда
```

Пример:

```ini
bind = SUPER, E, exec, [workspace 2 silent; float; move 0 0] kitty
```

### setprop

Свойства — это любые _динамические эффекты_ из [Правил окон](../Window-Rules#dynamic-effects).

Пример:

```sh
address:0x13371337 no_anim 1
address:0x13371337 no_max_size 0
address:0x13371337 opaque toggle
address:0x13371337 immediate unset
address:0x13371337 border_size relative -2
address:0x13371337 rounding_power relative 0.1
```

### Fullscreenstate

`fullscreenstate internal client`

Диспетчер `fullscreenstate` разделяет состояние полноэкранного режима, отслеживаемое Hyprland,
и состояние, сообщаемое клиентскому приложению.

`internal` — состояние, управляемое Hyprland.

`client` — состояние, получаемое приложением.

| Значение | Состояние | Описание |
| --- | --- | --- |
| -1 | Текущее | Сохраняет текущее состояние полноэкранного режима |
| 0 | Отсутствует | Окно занимает пространство, определённое текущей раскладкой |
| 1 | Максимизация | Окно занимает всё рабочее пространство, сохраняя отступы и панели |
| 2 | Полноэкранный режим | Окно занимает весь экран |
| 3 | Максимизация и полноэкранный режим | Состояние полноэкранного окна в режиме максимизации. Работает аналогично полноэкранному режиму |

Пример:

`fullscreenstate 2 0` — Переводит приложение в полноэкранный режим и сохраняет клиент в не-полноэкранном режиме.

Это можно использовать, чтобы предотвратить переход браузеров на базе Chromium в режим презентации при обнаружении полноэкранного режима.

`fullscreenstate 0 2` — Сохраняет окно в не-полноэкранном режиме, но клиентское приложение переходит в полноэкранный режим внутри окна.
