## Общее

Общая конфигурация монитора выглядит следующим образом:

```ini
monitor = имя, разрешение, позиция, масштаб
```

Распространённый пример:

```ini
monitor = DP-1, 1920x1080@144, 0x0, 1
```

Это сделает монитор на `DP-1` дисплеем с разрешением `1920x1080`,
частотой 144 Гц, смещением `0x0` от верхнего левого угла и масштабом 1 (без масштабирования).

Чтобы перечислить все доступные мониторы (активные и неактивные):

```bash
hyprctl monitors all
```

Мониторы размещаются на виртуальном «макете». `Позиция` — это положение
дисплея в макете в пикселях (рассчитывается от верхнего левого угла).

Например:

```ini
monitor = DP-1, 1920x1080, 0x0, 1
monitor = DP-2, 1920x1080, 1920x0, 1
```

заставит Hyprland разместить DP-1 _слева_ от DP-2, в то время как

```ini
monitor = DP-1, 1920x1080, 1920x0, 1
monitor = DP-2, 1920x1080, 0x0, 1
```

заставит Hyprland разместить DP-1 _справа_.

`Позиция` может содержать _отрицательные_ значения, поэтому приведённый выше пример также можно
записать как

```ini
monitor = DP-1, 1920x1080, 0x0, 1
monitor = DP-2, 1920x1080, -1920x0, 1
```

Hyprland использует инвертированную декартову систему по оси Y. Таким образом, отрицательная
координата Y размещает монитор выше, а положительная координата Y — ниже.

Например:

```ini
monitor = DP-1, 1920x1080, 0x0, 1
monitor = DP-2, 1920x1080, 0x-1080, 1
```

заставит Hyprland разместить DP-2 _выше_ DP-1, в то время как

```ini
monitor = DP-1, 1920x1080, 0x0, 1
monitor = DP-2, 1920x1080, 0x1080, 1
```

заставит Hyprland разместить DP-2 _ниже_.

> [!NOTE]
> Позиция рассчитывается с учётом масштабированного (и преобразованного) разрешения, то есть
> если вы хотите разместить ваш 4K-монитор с масштабом 2 слева от 1080p-монитора, вам следует
> использовать позицию `1920x0` для второго экрана (3840 / 2). Если монитор также повёрнут на 90 градусов
> (вертикально), следует использовать `1080x0`.

> [!WARNING]
> Мониторы не могут перекрываться. Это означает, что если заданные вами позиции приведут к перекрытию
> мониторов, вы получите предупреждение.

Оставление имени пустым определит резервное правило, которое будет использоваться, когда не совпадает
ни одно другое правило.

Существует несколько специальных значений для разрешения:

- `preferred` — использовать предпочтительное разрешение и частоту обновления дисплея.
- `highres` — использовать максимально поддерживаемое разрешение.
- `highrr` — использовать максимально поддерживаемую частоту обновления.
- `maxwidth` — использовать максимально широкое поддерживаемое разрешение.

Позиция также имеет несколько специальных значений:

- `auto` — позволить Hyprland определить позицию. По умолчанию каждый новый монитор размещается справа от существующих,
  используя верхний левый угол монитора как опорную точку.
- `auto-right/left/up/down` — разместить монитор справа/слева, выше или ниже других мониторов,
  также используя верхний левый угол каждого монитора как опорную точку.
- `auto-center-right/left/up/down` — разместить монитор справа/слева, выше или ниже других мониторов,
  но рассчитывать размещение от центра каждого монитора, а не от его верхнего левого угла.

_**Обратите внимание:**_ Хотя указание направления для первого монитора разрешено, это ничего не даёт,
и он будет размещён в точке (0,0). Кроме того, направление всегда рассчитывается от центра наружу,
поэтому вы можете указать сначала `auto-up`, затем `auto-left`, но мониторы слева будут находиться
просто слева от начала координат и выше него. Вы также можете указать повторяющиеся направления,
и мониторы будут продолжать размещаться в этом направлении.

Вы также можете использовать `auto` в качестве масштаба, чтобы позволить Hyprland определить масштаб за вас.
Это зависит от PPI монитора.

Рекомендуемое правило для быстрого подключения произвольных мониторов:

```ini
monitor = , preferred, auto, 1
```

Это автоматически разместит любой монитор, для которого не задано явное правило,
справа от остальных с его предпочтительным разрешением.

Для более точных правил можно также использовать описание выхода (см.
`hyprctl monitors` для дополнительных сведений). Если вывод команды `hyprctl monitors` выглядит
следующим образом:

```yaml
Monitor eDP-1 (ID 0):
        1920x1080@60.00100 at 0x0
        description: Chimei Innolux Corporation 0x150C (eDP-1)
        make: Chimei Innolux Corporation
        model: 0x150C
        [...]
```

то значение `description` до, но не включая имя порта `(eDP-1)`, можно
использовать для указания монитора:

```ini
monitor = desc:Chimei Innolux Corporation 0x150C, preferred, auto, 1.5
```

Не забудьте удалить `(имя_порта)`!

### Пользовательские модельные строки

Можно настроить пользовательскую модельную строку, изменив поле разрешения на модельную строку,
например:

```ini
monitor = DP-1, modeline 1071.101 3840 3848 3880 3920 2160 2263 2271 2277 +hsync -vsync, 0x0, 1
```

### Отключение монитора

Чтобы отключить монитор, используйте

```ini
monitor = имя, disable
```

> [!WARNING]
> Отключение монитора буквально удалит его из макета, переместив все окна
> и рабочие пространства на оставшиеся мониторы. Если вы хотите отключить монитор
> в стиле хранителя экрана (просто выключить дисплей), используйте диспетчер `dpms`
> [dispatcher](../Dispatchers).

## Пользовательская зарезервированная область

Зарезервированная область — это область, которая остаётся незанятой окнами в плиточной раскладке.
Если ваш рабочий процесс требует пользовательской зарезервированной области, вы можете добавить её с помощью:

```ini
monitor = имя, addreserved, ВВЕРХ, ВНИЗ, СЛЕВА, СПРАВА
```

Где `ВВЕРХ` `ВНИЗ` `СЛЕВА` `СПРАВА` — целые числа, то есть количество пикселей
зарезервированной области для добавления. Это значение добавляется к рассчитанной зарезервированной области
(например, панелей), но в конфигурации для каждого монитора можно использовать только одно такое правило.

## Дополнительные аргументы

Дополнительные аргументы можно комбинировать в конце правила монитора, например:

```ini
monitor = eDP-1, 2880x1800@90, 0x0, 1, transform, 1, mirror, DP-2, bitdepth, 10
```

Подробнее о каждом аргументе см. ниже.

### Зеркальные дисплеи

Чтобы зеркалировать дисплей, добавьте `, mirror, <ИМЯ>` в конец правила монитора, например:

```ini
monitor = DP-3, 1920x1080@60, 0x0, 1, mirror, DP-2
monitor = , preferred, auto, 1, mirror, DP-1
```

Имейте в виду, что при зеркалировании дисплеев изображение не будет «перерисовано» для второго
монитора, поэтому при зеркалировании 1080p-экрана на 4K-дисплей разрешение на 4K-дисплее
всё равно останется 1080p. Это также означает, что будут происходить сжатие и растяжение
при различающихся соотношениях сторон (например, 16:9 и 16:10).

### Поддержка 10 бит

Чтобы включить поддержку 10 бит для дисплея, добавьте `, bitdepth, 10` в конец правила монитора, например:

```ini
monitor = eDP-1, 2880x1800@90, 0x0, 1, bitdepth, 10
```

> [!WARNING]
> Цвета, зарегистрированные в Hyprland (например, цвет границы), _не_ поддерживают
> 10 бит.  
> Некоторые приложения _не_ поддерживают захват экрана при включённой 10-битной глубине цвета.

### Предустановки управления цветом

Добавьте `, cm, X`, чтобы изменить предустановку вывода sRGB по умолчанию:

```ini
monitor = eDP-1, 2880x1800@90, 0x0, 1, bitdepth, 10, cm, wide
```

```plain
auto    - srgb для 8bpc, wide для 10bpc, если поддерживается (рекомендуется)
srgb    - основные цвета sRGB (по умолчанию)
dcip3   - основные цвета DCI P3
dp3     - основные цвета Apple P3
adobe   - основные цвета Adobe RGB
wide    - широкая цветовая гамма, основные цвета BT2020
edid    - основные цвета из edid (известны неточностями)
hdr     - широкая цветовая гамма и функция передачи HDR PQ (экспериментально)
hdredid - то же, что и hdr, но с основными цветами из edid (экспериментально)
```

Полноэкранный HDR возможен без настройки `cm` hdr, если включена опция `render:cm_fs_passthrough`.

Используйте `sdrbrightness, B` и `sdrsaturation, S` для управления яркостью и насыщенностью SDR в режиме HDR.
Значения по умолчанию для обоих параметров — `1.0`. Типичное значение яркости должно находиться в диапазоне `1.0 ... 2.0`.

```ini
monitor = eDP-1, 2880x1800@90, 0x0, 1, bitdepth, 10, cm, hdr, sdrbrightness, 1.2, sdrsaturation, 0.98
```

Функция передачи по умолчанию, предполагаемая для использования на SDR-дисплее для sRGB-контента,
определяется параметром `, sdr_eotf, X`. Значение по умолчанию (`0`) следует настройке `render:cm_sdr_eotf`.
Это можно изменить на пошаговую sRGB с помощью `1` или на гамму 2.2 с помощью `2`.

### VRR

VRR для отдельного дисплея можно настроить, добавив `, vrr, X`, где `X` — режим из
[страницы переменных](../Variables).

## Поворот

Чтобы повернуть монитор, добавьте `, transform, X` в конец правила монитора,
где `X` соответствует номеру преобразования, например:

```ini
monitor = eDP-1, 2880x1800@90, 0x0, 1, transform, 1
```

Список преобразований:

```plain
0 -> нормальное (без преобразований)
1 -> 90 градусов
2 -> 180 градусов
3 -> 270 градусов
4 -> отражение
5 -> отражение + 90 градусов
6 -> отражение + 180 градусов
7 -> отражение + 270 градусов
```

## Monitor v2

Альтернативный синтаксис. `monitor = DP-1,1920x1080@144,0x0,1,transform,2` эквивалентно

```ini
monitorv2 {
  output = DP-1
  mode = 1920x1080@144
  position = 0x0
  scale = 1
  transform = 2
}
```

Флаг `disable` преобразуется в `disabled = true`, но остальные именованные настройки сохраняют свои имена:
`имя, значение` → `имя = значение` (например, `bitdepth,10` → `bitdepth = 10`).

Переопределения EDID и настройки SDR → HDR:

| имя | описание | тип |
|---|---|---|
| supports_wide_color | Принудительное включение поддержки широкой цветовой гаммы (0 — авто, 1 — включить, -1 — выключить) | int |
| supports_hdr | Принудительное включение поддержки HDR. Требует широкой цветовой гаммы (0 — авто, 1 — включить, -1 — выключить) | int |
| sdr_min_luminance | Минимальная яркость SDR, используемая для преобразования SDR → HDR. Установите 0.005 для точного соответствия чёрного цвета HDR | float |
| sdr_max_luminance | Максимальная яркость SDR. Может использоваться для регулировки общей яркости при преобразовании SDR → HDR. Разумный диапазон: 80–400. Желаемое значение, вероятно, находится между 200 и 250 | int |
| min_luminance | Минимальная яркость монитора | float |
| max_luminance | Максимально возможная яркость монитора | int |
| max_avg_luminance | Максимальная средняя яркость монитора для типичного кадра | int |

Примечание: эти значения могут быть переданы самому монитору и вызвать повышенный износ или другие повреждения,
если в его прошивке отсутствуют необходимые проверки безопасности.
